Docker Engine

1.网络拓扑图

宿主机（被分配的IP地址） -> NAT/端口转发（宿主机分配内部IP地址） -> 虚拟机，实际宿主机（192.168.65.0/24） -> iptables/DNAT（虚拟机分配内部IP地址） -> Container（172.x.x.x） -> 容器内服务再次请求占用容器的内网IP地址，以wireguard为例 -> wireguard（10.x.x.x /24）

补充：如果容器启动时为host模式，则iptables/DNAT（虚拟机分配内部IP地址） -> Container（172.x.x.x）部分不生效，转变为iptables/DNAT（虚拟机分配本机IP地址） -> Container（192.168.65.x）

3.Docker多级网络间的通信

场景 1：macOS 宿主机 <-> Docker 容器

最佳实践：端口映射 (Port Mapping)
不要尝试直接从 macOS ping 容器的 172.x.x.x IP（通常不通，因为 NAT 隔离）。
操作: 启动容器时映射端口。

bash

访问: 在 macOS 浏览器或终端访问 http://localhost:8080 或 http://127.0.0.1:8080。

原理: Docker Desktop 会自动将 macOS 的 localhost 流量转发到 VM，再转发到容器。

进阶：直接访问容器 IP (不推荐用于生产，仅限调试)

在某些配置下，macOS 可能无法直接路由到 172.17.0.x。如果必须直连：

1.确保容器端口已暴露（EXPOSE）。

2.可能需要配置 macOS 的路由表指向 VM 的 IP（复杂且不稳定，随 Docker 重启变化）。

替代方案: 使用 docker run –network host (在 Mac 上行为不同于 Linux，它实际上是让容器共享 VM 的网络栈，此时容器监听端口等同于 VM 监听端口，你仍需通过 localhost 访问)。

场景 3：外部局域网设备 <-> macOS 上的容器

如果你希望同一 WiFi 下的手机或其他电脑访问 Mac 上的 Docker 容器：

  1. 端口映射: 必须使用 -p 映射端口（如 -p 8080:80）。
  2. 防火墙: 确保 macOS 防火墙允许传入连接。
  3. 访问地址: 使用 macOS 的本机局域网 IP (如 192.168.1.50:8080)，而不是 localhost。
  • 注意: Docker Desktop 默认绑定的是 127.0.0.1。若要允许外部访问，需在 Docker Desktop 设置中勾选 “Allow the default Docker socket to be used (requires password)” 或者在启动时指定绑定地址（新版 Docker Desktop 有时限制绑定到 0.0.0.0 的行为，需在 Settings -> Advanced 中检查网络绑定设置）。

如果容器需要访问 Mac 上运行的数据库（如 Mac 本地安装的 MySQL）：

  • 特殊 DNS: Docker Desktop 提供了一个特殊的 DNS 名称 host.docker.internal。
  • 操作: 在容器内连接 host.docker.internal:3306 即可访问 Mac 宿主机的 3306 端口。
