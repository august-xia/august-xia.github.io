MacOS Docker Desktop部署wiregurad服务端

MacOS Docker部署wireguard

  1. 安装docker。
  2. 准备yaml配置文件。

  1. 启动文件。

    docker compose up -d

终端terminal 代码

ip转发部分（必须执行，小数点后为多选）

1.1 检查ip转发状态（开启/关闭）

sysctl net.inet.ip.forwarding

2.1 手动开启ip转发（重启后失效）

sudo sysctl -w net.inet.ip.forwarding=1

2.2 自动开启ip转发（开机自启动）

#1.创建或查看用于开启ip转发的脚本
sudo nano /Library/LaunchDaemons/com.enable.ipforward.plist

#2.编辑脚本
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.enable.ipforward</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/sbin/sysctl</string>
        <string>-w</string>
        <string>net.inet.ip.forwarding=1</string>
        <!-- 如需开启 IPv6 转发，添加下面一行 -->
        <!-- <string>net.inet6.ip6.forwarding=1</string> -->
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <false/>
    <key>User</key>
    <string>root</string>
</dict>
</plist>

#3.设置权限（必须，否则 launchd 无法加载）
sudo chown root:wheel /Library/LaunchDaemons/com.enable.ipforward.plist
sudo chmod 644 /Library/LaunchDaemons/com.enable.ipforward.plist

#4.加载配置（无需重启）
sudo launchctl load /Library/LaunchDaemons/com.enable.ipforward.plist

3.1 设置nat(ip转发)规则

#1.查看所有nat规则
sudo pfctl -s nat

#2.创建或查看nat的脚本
sudo nano /etc/pf.anchors/com.wireguard.nat

#3.1编辑nat规则（开启默认端口，范围为49152–65535）
nat on en1 from 10.13.13.0/24 to any -> (en1)
#3.2编辑nat规则（开启全部可用端口）
nat on en1 from 10.13.13.0/24 to any -> (en1) port 1024:65535

#4.启动nat规则
sudo pfctl -ef /etc/pf.anchors/com.wireguard.nat

其余部分（可选）

重要

监听抓包 所有网卡(any) 51820端口(51820) UDP(udp)流量

sudo tcpdump -n -i any udp port 51820

不重要

检查本机网口

route -n get default | grep interface

51820端口下的所有网络进程

lsof -i :51820

监听所有相关条目（可以进入docker文件夹执行）

netstat -an

512820端口下的所有相关条目

netstat -an | grep 51820

进入docker/wireguard bash专用

进入文件夹：docker exec -it wireguard /bin/sh

检查wireguard历史连接状况：wg show
